# 🎯 Tracking Platform — Роботизированная платформа слежения

Роботизированная поворотная платформа для автоматического слежения и предсказания траектории движущихся воздушных объектов (дроны, БПЛА, самолёты). Учебно-исследовательский DIY-проект.

---

## 📋 Состав системы

| Компонент | Модель | Назначение |
|-----------|--------|------------|
| Компьютер | Raspberry Pi 5 16 ГБ | Управление, баллистика, HUD |
| AI-ускоритель | AI HAT+ 2 (Hailo-10H, 40 TOPS) | Детекция YOLO26 на NPU |
| Камера | Arducam 5MP 1080p PTZ | Видео + оптический зум |
| Дальномер | Meskernel TS1224 (UART) | Лазерный замер дистанции |
| Моторы (2 шт.) | Cubemars AK80-64 (24V, MIT Mode) | Yaw/Pitch наведение |
| CAN-адаптер | CAN HAT для RPi (SocketCAN-совместимый) | CAN-шина для моторов |
| Джойстик | Logitech X56 HOTAS (правая ручка) | Ручное управление |
| Монитор | 9" автомобильный HDMI | Вывод видео с HUD |

---

## 🏗️ Архитектура (5 потоков)

```
Поток 1: Camera Thread (30 FPS)
  Arducam PTZ → camera_driver.py → кадры в буфер

Поток 2: Detection Thread (YOLO26 + BoT-SORT)
  Кадр → Hailo-10H NPU (40 TOPS) → детекция → BoT-SORT трекинг → TargetManager

Поток 3: LRF Thread (10 Гц)
  Meskernel TS1224 → UART → дистанция в SharedState

Поток 4: Motor Thread (100 Гц) ← ВЫСШИЙ ПРИОРИТЕТ
  MIT Mode → CAN → AK80-64 × 2 (velocity FF + gravity compensation)

Поток 5: Main Thread (30 FPS)
  Джойстик → State Machine → Баллистика (RK4 + G1) → HUD (OpenCV) → Монитор
```

## ⚡ Ключевые возможности

### Детекция и трекинг
- **YOLO26** (Ultralytics, последняя версия) на **Hailo-10H NPU** (40 TOPS)
- Встроенный **BoT-SORT** трекер (фильтр Калмана, Re-ID, CMC)
- **Классификация целей**: FPV-дрон, развед. дрон, ударный дрон, самолёт, вертолёт, птица
- **Приоритет угрозы**: FPV=10, ударный=9, ракета=9, птица=1

### Баллистический калькулятор
- **RK4** численное интегрирование (метод Рунге-Кутты 4-го порядка)
- **G1 drag-модель** Mayevski/Ingalls (18 диапазонов Маха)
- **Атмосфера**: температура, давление, влажность, высота (формула Бака)
- **Spin-drift** (гироскопический снос по формуле Litz)
- **Итеративный перехват** движущейся цели (8 итераций, сходимость < 0.5 мс)
- **2 профиля**: КОРД 12.7×108 (BC=0.670) и ПКТ 7.62×54R (BC=0.395)

### Управление моторами (MIT Mode)
- Все 5 параметров MIT: position, velocity_ff, kp, kd, torque_ff
- **Gravity compensation**: τ = m·g·L·cos(θ) для pitch-оси (7.3 Нм для КОРД)
- **Velocity feedforward**: компенсация инерции 40 кг при слежении
- **Динамические gains**: kp/kd меняются при стрельбе для гашения отдачи
- **Параллакс камера-LRF**: коррекция baseline (настраиваемый)
- **Boresight offsets**: таблица пристрелки с линейной интерполяцией

### Раздельное наведение (Split Aiming)
В АВТО-режиме при наличии баллистического решения:
- **Платформа (Cubemars)** наводит **ствол** на точку перехвата (lead point)
- **Камера PTZ (Arducam)** компенсирует разницу через Pan/Tilt сервоприводы, удерживая **цель в центре кадра**
- YOLO продолжает видеть цель, дальномер продолжает замерять дистанцию
- Оператору не нужно переводить перекрестие — ствол автоматически направлен в точку, куда полетит пуля
- При отсутствии Arducam SDK — fallback на виртуальный ромб (lead point на HUD)

### Боевой HUD-прицел (OpenCV, на русском)
Оптимизирован для быстрого принятия решений оператором (3 уровня информации):
- **Прицельный крест** с мил-дотами, цвет меняется по статусу (зелёный/красный)
- **Уголки bbox** цели + **тип цели крупно** с цветом по уровню угрозы (красный=FPV, оранжевый=развед., серый=птица)
- **Стрелка вектора движения** цели от центра bbox
- **Пульсирующая точка перехвата** (2 Гц, ромб, пунктирная линия, метка ToF)
- **Шкала дальности** (вертикальная, слева): зелёная зона поражения, серая вне зоны, треугольный маркер дистанции
- **Индикатор готовности** (нижний центр): **ГОТОВ** (пульсирует) / **ЗАМЕРЬ ДИСТ** / **ВНЕ ЗОНЫ** / **НЕ ГОТОВ**
- **Панель огневых данных** (правый верх): оружие, дистанция (цвет по зоне), ToF, скорость+направление (С/ЮВ/З), зум
- **Индикатор стрельбы**: мигающий ОГОНЬ [N выстрелов] + полоска отдачи
- **Шкала углов платформы** (нижний центр, вместо компаса — нет магнитометра)
- Статус режима (РУЧНОЙ/АВТО) и захвата цели (левый верх)

### Два режима
| Режим | Описание |
|-------|----------|
| **РУЧНОЙ** | Джойстик → скорость поворота |
| **АВТО** | PID-автослежение за захваченной целью |

### Управление джойстиком
| Элемент | Действие |
|---------|----------|
| Стик X/Y | Наведение платформы |
| Колёсико | Оптический зум |
| Триггер (указательный) | **ОГОНЬ** (активирует усиленные gains для гашения отдачи) |
| Кнопка A (большой палец) | АВАРИЙНАЯ ОСТАНОВКА |
| Кнопка B (большой палец) | Захват / сброс цели |
| Кнопка C | Замер дальности лазером |
| Кнопка D | Переключение КОРД ↔ ПКТ |
| Pinky (мизинец) | РУЧНОЙ ↔ АВТО |
| Hat Push | Возврат в центр |

## 📁 Структура проекта

```
tracking_platform/
├── main.py                         # Точка входа
├── config/
│   ├── app_config.py               # Типизированные dataclass-конфиги
│   └── settings.yaml               # Все настройки (моторы, баллистика, HUD...)
├── core/
│   ├── state_machine.py            # State Machine (Manual/Auto/E-STOP)
│   ├── platform_controller.py      # Главный контроллер (5 потоков)
│   ├── autotracker.py              # PID-автослежение
│   ├── ballistic_calculator.py     # RK4 + G1 drag + spin-drift + перехват
│   └── trajectory_predictor.py     # Предсказание траектории цели
├── drivers/
│   ├── cubemars_driver.py          # CAN MIT Mode (AK80-64)
│   ├── joystick_driver.py          # Logitech X56 HOTAS (pygame)
│   ├── camera_driver.py            # Arducam PTZ + зум
│   └── rangefinder_driver.py       # Meskernel TS1224 (UART)
├── perception/
│   ├── detector.py                 # YOLO26 (Hailo NPU / Ultralytics)
│   └── tracker.py                  # TargetManager (YOLO26 BoT-SORT)
├── ui/
│   └── hud_renderer.py             # HUD-прицел (OpenCV)
├── docs/
│   └── FULL_SETUP_GUIDE.md         # Полная инструкция по сборке
├── install.sh                      # Автоустановка всего ПО
└── requirements.txt                # Python-зависимости
```

## 🚀 Быстрый старт

```bash
# 1. Клонировать
git clone https://github.com/VladimirAndreev1985/tracking_platform.git
cd tracking_platform

# 2. Установить всё
bash install.sh

# 3. Перезагрузить (если CAN overlay добавлен впервые)
sudo reboot

# 4. Поднять CAN
./setup_can.sh

# 5. Запустить
./run.sh
```
